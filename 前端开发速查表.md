# 前端开发速查表 ⚡

## 🎯 EXE vs Web 快速对比

| 项目 | EXE桌面版 | Web网页版 |
|------|----------|----------|
| **后端地址** | `http://127.0.0.1:8000/api` | `https://local-show.microcall.com.cn/api` |
| **启动页** | 紫色Splash页面 | 直接登录页 |
| **环境变量** | 无 | `VITE_API_URL` + `VITE_SKIP_SPLASH=true` |
| **打包命令** | `快速重新打包前端.bat` | `重新打包Web前端.bat` |
| **测试方式** | 运行 `.exe` 文件 | 部署到服务器 |

## ✅ 代码规范

### 路由跳转（必须加 /app 前缀）

```typescript
// ✅ 正确
navigate('/app/appointments?patient_id=123');
navigate('/app/print-center?patient_id=456');
navigate('/app/dashboard');

// ❌ 错误（会404）
navigate('/appointments');
navigate('/print-center');
```

### API 请求（必须用 apiClient）

```typescript
// ✅ 正确
import { apiClient } from '@/api/client';
apiClient.get('/patients');
apiClient.post('/appointments', data);

// ❌ 错误（不会自动加 /api 前缀）
axios.get('http://127.0.0.1:8000/patients');
```

## 🚫 禁止修改

| 文件 | 原因 |
|------|------|
| `frontend/src/api/client.ts` | API配置，影响EXE和Web |
| `frontend/src/App.tsx` | 路由和环境检测 |
| `frontend/src/pages/Splash.tsx` | EXE启动页 |
| `frontend/src-tauri/` | Tauri配置 |

## 🐛 快速调试

### 1. 查看环境信息
访问：`/debug` 路由

### 2. 检查API请求
按 `F12` → Network 标签 → 查看请求URL

### 3. 常见错误

| 错误 | 原因 | 解决 |
|------|------|------|
| 404 Not Found | 路由缺少 `/app` | 加上 `/app` 前缀 |
| API 404 | 缺少 `/api` 前缀 | 使用 `apiClient` |
| EXE无Splash | 环境检测错误 | 检查 `/debug` 页面 |
