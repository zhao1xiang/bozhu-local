# 患者备注功能说明

## 版本信息
- 版本：v2.2.2
- 更新日期：2026-03-11
- 功能：新增患者备注字段

## 功能概述
在患者管理中新增"备注"字段，用于记录患者的额外信息、特殊说明等。

## 功能详情

### 1. 表单位置
- 位置：患者新增/编辑表单的最下方
- 类型：多行文本输入框
- 字数限制：500字
- 是否必填：否（选填）

### 2. 字段特性
- 字段名：remarks
- 数据类型：TEXT
- 默认值：NULL
- 显示：带字数统计

### 3. 使用场景
- 记录患者特殊情况
- 备注治疗注意事项
- 记录沟通要点
- 其他需要说明的信息

### 4. 批量导入支持
- Excel模板已更新，新增"备注"列
- 位置：第17列（最后一列）
- 导入时自动处理备注信息

## 数据库变更

### 新增字段
```sql
ALTER TABLE patient ADD COLUMN remarks TEXT;
```

### 字段说明
- 表名：patient
- 字段名：remarks
- 类型：TEXT
- 描述：备注信息
- 可空：是

## 前端变更

### 1. 类型定义更新
文件：`frontend/src/types/index.ts`
```typescript
export interface Patient {
  // ... 其他字段
  remarks?: string;
}
```

### 2. 表单更新
文件：`frontend/src/pages/Patients.tsx`
- 新增备注输入框
- 位置：表单最下方
- 组件：Input.TextArea
- 配置：3行，最多500字，显示字数

## 后端变更

### 1. 模型更新
文件：`backend/models/patient.py`
```python
remarks: Optional[str] = Field(default=None, description="备注")
```

### 2. 批量导入更新
文件：`backend/routers/patients.py`
- 读取第17列（备注）
- 保存到数据库

### 3. 自动迁移更新
文件：`backend/auto_migrate.py`
- 添加remarks字段到迁移列表
- 启动时自动检测并添加

## 迁移说明

### 自动迁移
程序启动时会自动检测并添加remarks字段，无需手动操作。

### 手动迁移
如需手动迁移，运行：
```bash
python backend/migrate_add_remarks.py
```

### 数据兼容性
- 旧数据：remarks字段为NULL
- 新数据：可选填写备注
- 完全向后兼容
