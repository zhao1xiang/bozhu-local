# 构建32位版本指南

## 前提条件

### 1. 安装32位Rust工具链

```bash
# 安装32位工具链
rustup target add i686-pc-windows-msvc
```

### 2. 安装32位Python（用于后端）

- 下载Python 3.11 32位版本：https://www.python.org/downloads/
- 安装时选择32位版本
- 记录安装路径，例如：`C:\Python311-32\`

### 3. 安装Visual Studio Build Tools

确保安装了32位的C++构建工具：
- 下载：https://visualstudio.microsoft.com/downloads/
- 选择"Desktop development with C++"
- 确保包含32位工具

## 构建步骤

### 方法1：使用自动化脚本（推荐）

我会创建一个自动化脚本来构建32位版本。

### 方法2：手动构建

#### 步骤1：构建32位后端

```bash
# 使用32位Python
C:\Python311-32\python.exe -m pip install pyinstaller
C:\Python311-32\python.exe -m PyInstaller backend/build_backend.spec --clean
```

#### 步骤2：构建32位前端

```bash
cd frontend
npm run build
```

#### 步骤3：构建32位Tauri应用

```bash
cd frontend
npm run tauri build -- --target i686-pc-windows-msvc
```

## 注意事项

### 性能考虑
- 32位应用内存限制为2GB
- 性能可能不如64位版本
- 建议仅在必要时使用

### 兼容性
- ✅ Windows 7 32位（需要SP1）
- ✅ Windows 8/8.1 32位
- ✅ Windows 10 32位
- ⚠️ 需要安装WebView2 Runtime（32位版本）

### WebView2要求
- 32位系统需要安装32位的WebView2 Runtime
- 下载地址：https://go.microsoft.com/fwlink/p/?LinkId=2124703
- 选择"x86"版本

## 构建输出

32位版本将输出到：
```
frontend/src-tauri/target/i686-pc-windows-msvc/release/bundle/
```

## 测试建议

1. 在32位虚拟机中测试
2. 验证所有功能正常
3. 检查内存使用情况
4. 测试打印功能

## 已知限制

1. **内存限制**：32位应用最多使用2GB内存
2. **性能较差**：相比64位版本性能下降约20-30%
3. **维护成本**：需要维护两个版本的构建流程

## 建议

对于新客户，强烈建议：
1. 升级到64位操作系统
2. 或使用Web版本（无系统限制）

仅在客户确实无法升级系统时提供32位版本。
