版本 v2.1.8 更新日志
==================

发布日期：2026-03-05

【新增功能】

1. 患者诊断"其他"选项
   - 在患者管理中，诊断选择"其他"时会显示文本输入框
   - 可以输入具体的诊断说明
   - 数据会保存到 diagnosis_other 字段

2. 视力字段改造
   - 患者表单：分为"裸眼视力"和"矫正视力"两个区块
   - 预约表单：使用Card组件展示，更清晰
   - 新增字段：left_vision_corrected, right_vision_corrected

3. 药物"其他"选项
   - 患者管理和预约管理都支持
   - 选择"其他"时显示文本输入框
   - 可以输入具体的药物名称

4. 注射周期配置化
   - 系统配置中新增"前4针注射间隔"配置项
   - 默认1个月，可设置1-12个月
   - 第5针及以后保持固定规则（2个月、3个月、4个月）
   - 所有预约生成逻辑都使用配置的间隔

5. 打印电话号码配置
   - 系统配置中新增"打印联系电话"配置项
   - 打印时会显示配置的电话号码
   - 显示在打印模板的右下角

【优化改进】

1. 数据库自动迁移
   - 启动时自动检测并添加新字段
   - 自动初始化系统配置
   - 自动备份数据库

2. 后端启动优化
   - 改进 uvicorn 启动方式
   - 使用 Server 类而不是 run() 函数
   - 明确指定事件循环类型
   - 更好的错误日志

3. 用户体验优化
   - 表单布局更清晰
   - 条件渲染更流畅
   - 配置项更直观

【修复问题】

1. 修复后端启动卡住的问题
   - 改用 uvicorn.Server 类启动
   - 明确指定 asyncio 事件循环

2. 修复变量重复声明错误
   - 清理了重复的 intervalMonths 声明

【数据库变更】

新增字段：
- patient.diagnosis_other (TEXT) - 诊断其他说明
- patient.drug_type_other (TEXT) - 药物其他说明
- patient.left_vision_corrected (REAL) - 左眼矫正视力
- patient.right_vision_corrected (REAL) - 右眼矫正视力
- appointment.drug_name_other (TEXT) - 药品其他说明

新增系统配置：
- injection_interval_first_4 = '1' - 前4针注射间隔（月）
- print_phone_number = '13608685716' - 打印联系电话

【升级说明】

从 v2.1.7 升级到 v2.1.8：
1. 直接替换 backend_server.exe
2. 替换 frontend 文件夹
3. 启动时会自动升级数据库
4. 无需手动操作

【注意事项】

1. 首次启动会自动升级数据库结构
2. 会自动创建数据库备份
3. 所有新字段都是可选的，不影响现有功能
4. 建议在升级前手动备份 database.db 文件
