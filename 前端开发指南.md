# 眼科注射预约系统 - 前端开发指南

## 📋 项目概述

本项目是一个**双端应用**，同一套前端代码需要支持：
1. **EXE桌面版**（基于 Tauri）- 给医院内部使用
2. **Web网页版**（部署到服务器）- 给外部演示使用

## 🎯 核心区别

### 1. 运行环境

| 特性 | EXE桌面版 | Web网页版 |
|------|----------|----------|
| 运行方式 | Windows应用程序 | 浏览器访问 |
| 后端地址 | `http://127.0.0.1:8000/api` | `https://local-show.microcall.com.cn/api` |
| 启动页面 | 显示 Splash 加载页 | 直接进入登录页 |
| 后端检测 | 需要检测后端启动 | 跳过后端检测 |
| 环境变量 | 无需设置 | 需要设置 `VITE_API_URL` 和 `VITE_SKIP_SPLASH` |

### 2. 环境变量配置

**EXE版本（默认）：**
```typescript
// 不设置任何环境变量
VITE_API_URL = undefined  // 使用默认值 http://127.0.0.1:8000/api
VITE_SKIP_SPLASH = undefined  // 显示 Splash 页面
```

**Web版本（打包时设置）：**
```bash
VITE_API_URL=https://local-show.microcall.com.cn/api
VITE_SKIP_SPLASH=true
```

## 🔧 关键代码说明

### 1. API 配置 (`frontend/src/api/client.ts`)

```typescript
// ⚠️ 重要：默认值必须包含 /api 前缀
const API_BASE_URL = import.meta.env.VITE_API_URL || 'http://127.0.0.1:8000/api';
```

**注意事项：**
- EXE版本使用默认值：`http://127.0.0.1:8000/api`
- Web版本通过环境变量覆盖：`https://local-show.microcall.com.cn/api`
- **所有API路径都会自动加上这个前缀**

### 2. 环境检测 (`frontend/src/App.tsx`)

```typescript
// 检测是否在 Tauri 环境中（EXE）
const isTauri = () => {
  // 方法1: 检查 __TAURI__ API
  if (typeof window !== 'undefined' && window.__TAURI__) {
    return true;
  }
  
  // 方法2: 检查 URL 协议
  if (typeof window !== 'undefined') {
    const protocol = window.location.protocol;
    const hostname = window.location.hostname;
    
    if (protocol === 'tauri:' || hostname === 'tauri.localhost') {
      return true;
    }
  }
  
  // 方法3: 检查是否不是开发服务器
  if (typeof window !== 'undefined') {
    const href = window.location.href;
    if (!href.startsWith('http://localhost:') && !href.startsWith('https://')) {
      return true;
    }
  }
  
  return false;
};

// 是否显示 Splash 页面
const shouldShowSplash = () => {
  // 如果设置了环境变量跳过检测
  if (import.meta.env.VITE_SKIP_SPLASH === 'true') {
    return false;
  }
  // 否则只在 Tauri 环境中显示
  return isTauri();
};
```

**注意事项：**
- EXE版本会显示 Splash 页面，检测后端启动
- Web版本跳过 Splash，直接进入登录页
- **不要修改这个检测逻辑，除非你完全理解它的作用**

### 3. 路由配置 (`frontend/src/App.tsx`)

```typescript
// 路由结构
/ (根路径)
├── / → Splash页面（EXE）或跳转到 /login（Web）
├── /login → 登录页面
├── /debug → 调试信息页面
└── /app → 主应用（需要登录）
    ├── /app/dashboard → 仪表盘
    ├── /app/patients → 患者管理
    ├── /app/appointments → 预约管理
    ├── /app/daily-work → 今日预约
    ├── /app/print-center → 打印中心
    └── /app/system-config → 系统配置
```

**⚠️ 重要：路由跳转规则**
```typescript
// ✅ 正确：应用内页面跳转必须加 /app 前缀
navigate('/app/appointments?patient_id=123');
navigate('/app/print-center?patient_id=456');

// ❌ 错误：缺少 /app 前缀会导致 404
navigate('/appointments?patient_id=123');  // 错误！
navigate('/print-center?patient_id=456');  // 错误！

// ✅ 正确：顶级路由不需要 /app 前缀
navigate('/login');  // 登录页
navigate('/debug');  // 调试页
```

## 📝 开发注意事项

### 1. 修改代码时的检查清单

- [ ] 路由跳转是否包含 `/app` 前缀？
- [ ] API请求是否使用 `apiClient`？（不要直接用 axios）
- [ ] 是否修改了环境检测逻辑？（需要测试EXE和Web两个版本）
- [ ] 是否修改了 API 配置？（需要确保默认值正确）

### 2. 常见错误

**错误1：路由跳转缺少 /app 前缀**
```typescript
// ❌ 错误
navigate('/appointments');

// ✅ 正确
navigate('/app/appointments');
```

**错误2：直接修改 API 地址**
```typescript
// ❌ 错误：硬编码API地址
axios.get('http://127.0.0.1:8000/api/patients');

// ✅ 正确：使用 apiClient
apiClient.get('/patients');  // 会自动加上 baseURL
```

**错误3：修改环境检测导致EXE无法显示Splash**
```typescript
// ❌ 错误：强制跳过Splash
const shouldShowSplash = () => false;

// ✅ 正确：根据环境判断
const shouldShowSplash = () => {
  if (import.meta.env.VITE_SKIP_SPLASH === 'true') {
    return false;
  }
  return isTauri();
};
```

## 🚀 开发流程

### 1. 本地开发

**步骤1：启动后端**
```bash
cd backend
python run_server.py
```

验证后端启动：访问 http://127.0.0.1:8000/api/health

**步骤2：启动前端**
```bash
cd frontend
npm install
npm run dev
```

访问：`http://localhost:5173`

**注意：**
- 本地开发时，后端和前端需要分别启动（两个终端窗口）
- 本地开发环境不会显示 Splash 页面（因为不是 Tauri 环境）
- 后端默认端口：8000，前端默认端口：5173

### 2. 测试EXE版本

```bash
# 快速打包（只打包前端和Tauri）
快速重新打包前端.bat

# 完整打包（包括后端）
完整打包流程-v2.1.1.bat
```

测试文件：`dist-package/眼科注射预约系统.exe`

**测试要点：**
- [ ] 启动时是否显示紫色 Splash 页面？
- [ ] Splash 页面是否检测后端启动？
- [ ] 登录是否正常？
- [ ] 所有路由跳转是否正常？

### 3. 测试Web版本

```bash
# 打包Web前端
重新打包Web前端.bat
```

构建产物：`frontend/dist/`

**测试要点：**
- [ ] 是否直接进入登录页（跳过Splash）？
- [ ] API请求是否指向正确的服务器地址？
- [ ] 所有路由跳转是否正常？

## 🐛 调试技巧

### 1. 使用调试页面

访问 `/debug` 路由可以查看环境信息：
- Tauri API 是否加载
- 当前 URL 协议和主机名
- 环境变量配置
- 是否应该显示 Splash

### 2. 浏览器控制台

在EXE或Web中按 `F12` 打开开发者工具，查看：
- 网络请求（Network）- 检查API请求地址
- 控制台（Console）- 查看错误信息
- 应用（Application）- 查看本地存储

### 3. 常见问题排查

**问题：EXE不显示Splash页面**
1. 打开 `/debug` 页面
2. 检查 "Tauri API" 是否已加载
3. 检查 "VITE_SKIP_SPLASH" 是否为 "未设置"

**问题：API请求404**
1. 打开浏览器开发者工具 → Network
2. 查看请求的完整URL
3. 确认是否包含 `/api` 前缀

**问题：路由跳转404**
1. 检查跳转代码是否包含 `/app` 前缀
2. 查看浏览器地址栏的URL
3. 对比路由配置表

## 🔒 重要提醒

### 不要修改的文件

除非你完全理解它们的作用，否则**不要修改**以下文件：

1. **`frontend/src/api/client.ts`** - API配置
   - 修改可能导致EXE或Web版本无法连接后端

2. **`frontend/src/App.tsx`** - 路由和环境检测
   - 修改可能导致EXE不显示Splash或路由错误

3. **`frontend/src/pages/Splash.tsx`** - 启动页面
   - 修改可能导致EXE启动异常

4. **`frontend/src-tauri/`** - Tauri配置
   - 修改可能导致EXE打包失败

### 可以安全修改的内容

✅ 页面UI和样式
✅ 业务逻辑
✅ 表单验证
✅ 数据展示
✅ 组件封装

**但是：**
- 路由跳转必须使用 `navigate('/app/...')`
- API请求必须使用 `apiClient.get/post/put/delete`
