# 后端环境搭建教程

本教程帮助前端工程师在本地搭建后端开发环境，用于前端开发和调试。

## 📋 前置要求

- Windows 10/11 操作系统
- 管理员权限（用于安装软件）

## 🔧 步骤1：安装 Python

### 1.1 下载 Python

访问 Python 官网下载页面：
- 官网：https://www.python.org/downloads/
- 推荐版本：**Python 3.11** 或更高版本

**下载链接：**
- 直接下载：https://www.python.org/ftp/python/3.11.9/python-3.11.9-amd64.exe

### 1.2 安装 Python

1. 双击下载的安装包
2. **⚠️ 重要：勾选 "Add Python to PATH"**（必须勾选！）
3. 点击 "Install Now"
4. 等待安装完成

### 1.3 验证安装

打开命令提示符（CMD）或 PowerShell，输入：

```bash
python --version
```

应该显示类似：`Python 3.11.9`

如果显示 `'python' 不是内部或外部命令`，说明没有正确添加到 PATH，需要重新安装并勾选 "Add Python to PATH"。

## 📦 步骤2：安装后端依赖

### 2.1 进入后端目录

```bash
cd backend
```

### 2.2 安装依赖包

```bash
pip install -r requirements.txt
```

这个命令会安装所有必需的 Python 包，包括：
- FastAPI - Web 框架
- Uvicorn - ASGI 服务器
- SQLModel - 数据库 ORM
- 其他依赖...

**安装时间：** 约 2-5 分钟（取决于网络速度）

### 2.3 验证安装

检查关键包是否安装成功：

```bash
pip show fastapi
pip show uvicorn
```

应该显示包的详细信息。

## 🚀 步骤3：启动后端服务

### 3.1 启动命令

在 `backend` 目录下，运行：

```bash
python run_server.py
```

### 3.2 验证启动成功

看到以下输出说明启动成功：

```
INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
INFO:     Started reloader process
INFO:     Started server process
INFO:     Waiting for application startup.
INFO:     Application startup complete.
```

### 3.3 测试后端接口

打开浏览器，访问以下地址：

1. **健康检查：** http://127.0.0.1:8000/api/health
   - 应该返回：`{"status":"ok","message":"Backend is ready"}`

2. **API 文档：** http://127.0.0.1:8000/docs
   - 可以看到所有 API 接口的文档和测试界面

3. **根路径：** http://127.0.0.1:8000/
   - 应该返回：`{"message":"Welcome to Eye Injection Appointment System API"}`

## 🎯 步骤4：启动前端（连接本地后端）

### 4.1 打开新的终端窗口

**重要：** 保持后端终端运行，打开一个新的终端窗口。

### 4.2 启动前端

```bash
cd frontend
npm install
npm run dev
```

### 4.3 访问前端

浏览器访问：http://localhost:5173

前端会自动连接到本地后端（`http://127.0.0.1:8000/api`）

## 🐛 常见问题

### 问题1：pip 命令不存在

**错误信息：**
```
'pip' 不是内部或外部命令
```

**解决方法：**
1. 重新安装 Python，确保勾选 "Add Python to PATH"
2. 或者使用：`python -m pip install -r requirements.txt`

### 问题2：端口 8000 被占用

**错误信息：**
```
[Errno 10048] error while attempting to bind on address ('127.0.0.1', 8000)
```

**解决方法：**

**方法1：查找并关闭占用端口的程序**
```bash
# 查找占用端口的进程
netstat -ano | findstr :8000

# 记下最后一列的 PID，然后关闭进程
taskkill /PID <PID> /F
```

**方法2：修改后端端口**

编辑 `backend/run_server.py`，将端口改为其他值（如 8001）：
```python
uvicorn.run(
    "main:app",
    host="127.0.0.1",
    port=8001,  # 改为 8001
    log_level="info",
    access_log=True
)
```

然后修改前端 API 配置 `frontend/src/api/client.ts`：
```typescript
const API_BASE_URL = import.meta.env.VITE_API_URL || 'http://127.0.0.1:8001/api';
```

### 问题3：依赖安装失败

**错误信息：**
```
ERROR: Could not find a version that satisfies the requirement...
```

**解决方法：**

**方法1：升级 pip**
```bash
python -m pip install --upgrade pip
```

**方法2：使用国内镜像源**
```bash
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 问题4：数据库文件不存在

**错误信息：**
```
[WARN] 数据库文件不存在: database.db
```

**解决方法：**

不用担心，后端会自动创建数据库文件。首次启动时会自动：
1. 创建 `database.db` 文件
2. 创建所有数据表
3. 创建默认管理员账号（admin/admin）

### 问题5：前端无法连接后端

**错误信息：**
```
Network Error
ERR_CONNECTION_REFUSED
```

**检查清单：**
1. [ ] 后端是否正在运行？（检查后端终端）
2. [ ] 后端端口是否是 8000？
3. [ ] 访问 http://127.0.0.1:8000/api/health 是否正常？
4. [ ] 前端 API 配置是否正确？（检查 `frontend/src/api/client.ts`）

## 📚 后端项目结构

了解后端项目结构，方便调试：

```
backend/
├── main.py                 # FastAPI 主文件，定义所有路由

├── models/               # 数据模型
│   ├── patient.py        # 患者模型
│   ├── appointment.py    # 预约模型
│   └── user.py          # 用户模型
└── routers/             # API 路由
    ├── patients.py      # 患者相关 API
    ├── appointments.py  # 预约相关 API
    ├── auth.py         # 认证相关 API
    └── ...
```

## 💡 开发技巧

### 1. 自动重启（开发模式）

如果需要修改后端代码并自动重启，使用：

```bash
cd backend
uvicorn main:app --host 127.0.0.1 --port 8000 --reload
```

`--reload` 参数会在代码修改后自动重启服务器。

### 2. 查看日志

后端日志会输出到：
- 终端（实时查看）
- `backend/logs/backend.log` 文件

### 3. 重置数据库

如果需要重置数据库（清空所有数据）：

```bash
cd backend
# 删除数据库文件
del database.db

# 重新启动后端，会自动创建新数据库
python run_server.py
```

### 4. 修改默认管理员密码

编辑 `backend/main.py`，找到 `on_startup` 函数：

```python
admin_user = User(username="admin", hashed_password=get_password_hash("admin"))
```

将 `"admin"` 改为你想要的密码。
