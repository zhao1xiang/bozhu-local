## 清理步骤建议

### 第一步：删除临时文件（安全）
```bash
# 删除临时构建目录
rm -rf temp-build/
rm -rf build/
rm -rf dist/

# 删除系统文件
rm .DS_Store
```

### 第二步：整理文档（建议）
```bash
# 创建文档目录
mkdir -p docs/fixes docs/build docs/user

# 移动修复报告
mv *修复*.md *错误*.md *兼容性*.md *字段*.md docs/fixes/

# 移动打包说明
mv *打包*.md docs/build/

# 移动用户文档
mv *说明*.md UserManual*.md UserManual.pdf docs/user/
```

### 第三步：整理工具脚本（建议）
```bash
# 创建工具目录
mkdir -p tools

# 移动工具脚本
mv *工具*.py *字段*.py tools/
```

### 第四步：删除旧脚本（可选）
```bash
# 删除旧版本的批处理脚本
rm *-v2.2.2.bat
rm test-backend.bat
rm backend_server.spec
```

## 注意事项

1. **备份重要文件**：在删除之前，建议先备份整个项目
2. **确认不再使用**：确保要删除的文件确实不再需要
3. **更新 .gitignore**：添加临时文件和构建目录到 .gitignore
4. **保留当前版本**：`simple-web-package-win7-v2.2.3/` 和 `web-dist-v2.2.3/` 是当前版本，必须保留
5. **测试数据库**：`database.db` 如果包含测试数据，建议保留
