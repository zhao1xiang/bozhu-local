# 矫正视力字段保存修复说明 - v2.2.3 (最终版)

## 修复时间
2026-03-13

## 问题描述
预约时填写的矫正视力（左眼矫正视力、右眼矫正视力）没有保存到数据库，导致编辑预约时无法回显这些字段的值。

## 问题原因（两个层面）

### 1. 后端模型缺少字段定义
`backend/models/appointment.py` 的 `AppointmentBase` 模型中根本没有定义矫正视力字段：
- 只有 `pre_op_vision_left` (左眼裸眼视力)
- 只有 `pre_op_vision_right` (右眼裸眼视力)
- 缺少 `pre_op_vision_left_corrected` (左眼矫正视力)
- 缺少 `pre_op_vision_right_corrected` (右眼矫正视力)

### 2. 前端保存逻辑缺少字段
`frontend/src/pages/Appointments.tsx` 的 `saveAppointment` 函数中，`commonFields` 对象也没有包含矫正视力字段。

### 3. 数据库兼容性检查缺少字段
`backend/database_compatibility.py` 的 `check_missing_fields` 函数中，预约表的必需字段列表也没有包含矫正视力字段。

## 解决方案

### 1. 后端模型添加字段
在 `backend/models/appointment.py` 中添加矫正视力字段：

```python
class AppointmentBase(SQLModel):
    # ... 其他字段 ...
    pre_op_vision_left: Optional[str] = None # 左眼术前视力（裸眼）
    pre_op_vision_right: Optional[str] = None # 右眼术前视力（裸眼）
    pre_op_vision_left_corrected: Optional[str] = None # 左眼术前视力（矫正）
    pre_op_vision_right_corrected: Optional[str] = None # 右眼术前视力（矫正）
    treatment_phase: Optional[str] = Field(default=None, description="治疗周期")
```

### 2. 前端保存逻辑添加字段
在 `frontend/src/pages/Appointments.tsx` 的 `saveAppointment` 函数中添加：

```typescript
const commonFields = {
  // ... 其他字段 ...
  pre_op_vision_left: values.pre_op_vision_left ? String(values.pre_op_vision_left) : '',
  pre_op_vision_right: values.pre_op_vision_right ? String(values.pre_op_vision_right) : '',
  pre_op_vision_left_corrected: values.pre_op_vision_left_corrected ? String(values.pre_op_vision_left_corrected) : '',
  pre_op_vision_right_corrected: values.pre_op_vision_right_corrected ? String(values.pre_op_vision_right_corrected) : '',
  // ... 其他字段 ...
};
```

### 3. 数据库兼容性检查添加字段
在 `backend/database_compatibility.py` 的 `check_missing_fields` 函数中添加：

```python
'appointment': [
    # ... 其他字段 ...
    ('pre_op_vision_left', 'TEXT', ''),
    ('pre_op_vision_right', 'TEXT', ''),
    ('pre_op_vision_left_corrected', 'TEXT', ''),
    ('pre_op_vision_right_corrected', 'TEXT', ''),
    ('treatment_phase', 'TEXT', ''),
]
```

## 修改的文件
1. `backend/models/appointment.py` - 添加矫正视力字段定义
2. `frontend/src/pages/Appointments.tsx` - 在 `saveAppointment` 函数的 `commonFields` 中添加矫正视力字段
3. `backend/database_compatibility.py` - 在必需字段列表中添加矫正视力字段

## 影响范围
- 新建预约：矫正视力字段现在会正确保存到数据库
- 编辑预约：矫正视力字段现在会正确回显
- 批量创建预约：矫正视力字段会应用到所有批量创建的预约
- 数据库迁移：启动时会自动添加缺失的矫正视力字段

## 数据库字段
预约表（appointment）中的矫正视力字段：
- `pre_op_vision_left_corrected` - TEXT 类型
- `pre_op_vision_right_corrected` - TEXT 类型

这两个字段会在系统启动时自动添加（如果不存在）。

## 注意事项
1. 矫正视力字段支持字符串输入（数字和文字，如 "0.8" 或 "手动/光感"）
2. 保存时会自动转换为字符串类型，避免类型错误
3. 如果字段为空，会保存为空字符串而不是 null
4. 老数据库会在启动时自动添加这两个字段
5. 需要重启后端服务才能生效
