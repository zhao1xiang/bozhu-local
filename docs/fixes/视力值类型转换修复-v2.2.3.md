# 视力值类型转换修复 - v2.2.3

## 🎯 问题描述

用户在新增患者时输入视力值（如1.2, 1.3, 1.4, 1.5），虽然使用的是Input组件（文本输入），但在某些情况下这些值可能被JavaScript解析为数字类型，导致在创建预约时出现类型不匹配错误。

### 问题场景
1. 新增患者，输入视力值：1.2, 1.3, 1.4, 1.5
2. 保存患者成功
3. 创建预约时，从患者信息带入视力值
4. 提交预约时报错（类型验证失败）

### 根本原因
- 前端虽然使用Input组件，但JavaScript可能将纯数字字符串解析为number类型
- 后端模型期望的是string类型
- 数据传递过程中没有显式转换为字符串

## ✅ 解决方案

### 1. 患者保存时强制转换
在 `frontend/src/pages/Patients.tsx` 的 `savePatient` 函数中：

```typescript
const patientData = {
  ...values,
  diagnosis: Array.isArray(values.diagnosis) ? values.diagnosis.join(',') : values.diagnosis || '',
  drug_type: Array.isArray(values.drug_type) ? values.drug_type.join(',') : values.drug_type || '',
  // 确保视力值始终是字符串类型
  left_vision: values.left_vision ? String(values.left_vision) : '',
  right_vision: values.right_vision ? String(values.right_vision) : '',
  left_vision_corrected: values.left_vision_corrected ? String(values.left_vision_corrected) : '',
  right_vision_corrected: values.right_vision_corrected ? String(values.right_vision_corrected) : '',
};
```

### 2. 预约保存时强制转换
在 `frontend/src/pages/Appointments.tsx` 的 `saveAppointment` 函数中：

```typescript
const commonFields = {

  pre_op_vision_left: values.pre_op_vision_left ? String(values.pre_op_vision_left) : '',
  pre_op_vision_right: values.pre_op_vision_right ? String(values.pre_op_vision_right) : '',
  treatment_phase: values.treatment_phase,
  // ... 其他字段
```

## 🔍 技术细节

### JavaScript类型转换
```javascript
// 问题场景
const input = "1.2";  // 用户输入
const value = 1.2;    // 可能被解析为数字

// 解决方案
const stringValue = String(value);  // "1.2" - 始终是字符串
```

### 处理的字段
**患者表**：
- `left_vision` - 左眼裸眼视力
- `right_vision` - 右眼裸眼视力
- `left_vision_corrected` - 左眼矫正视力
- `right_vision_corrected` - 右眼矫正视力

**预约表**：
- `pre_op_vision_left` - 左眼术前视力
- `pre_op_vision_right` - 右眼术前视力

### 转换规则
```typescript
// 有值时转换为字符串
value ? String(value) : ''

// 示例
String(1.2)      → "1.2"
String(1.3)      → "1.3"
String("手动")   → "手动"
String(null)     → "null" (不推荐)
null ? String(null) : '' → ""  (推荐)
```
