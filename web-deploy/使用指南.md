# Web 演示环境使用指南

## 快速开始

### 方式一：一键启动（推荐用于演示）

1. 首次部署，运行：
```bash
deploy.bat
```

2. 后续启动，运行：
```bash
quick-start.bat
```

3. 浏览器会自动打开 http://localhost:8080

### 方式二：手动启动

1. 启动后端：
```bash
cd backend
.venv\Scripts\activate
python main.py
```

2. 启动前端（新开一个终端）：
```bash
cd frontend\dist
python -m http.server 8080
```

3. 访问 http://localhost:8080

## 部署到生产服务器

### 使用 Nginx（推荐）

1. 运行部署脚本：
```bash
deploy.bat
```

2. 配置 Nginx：
```bash
# 复制配置文件
copy nginx.conf C:\nginx\conf\sites-available\eyecare.conf

# 修改配置文件中的路径和域名
# 重启 Nginx
nginx -s reload
```

3. 配置后端为系统服务（Windows）：
   - 使用 NSSM 或 Windows 服务管理器
   - 或使用 PM2（需要安装 Node.js）

### 使用 Docker（推荐用于 Linux）

```bash
docker-compose up -d
```

## 配置说明

### 前端配置

修改 API 地址（构建前）：

1. 编辑 `.env.production` 文件：
```
VITE_API_URL=https://your-api-domain.com/api
```

2. 重新构建：
```bash
cd ../frontend
npm run build
```

### 后端配置

后端配置在 `backend/main.py` 中：

- 端口：默认 8000
- CORS：默认允许所有来源（生产环境建议限制）
- 数据库：SQLite，位于 `backend/database.db`

## 与桌面版的区别

| 特性 | 桌面版（Tauri） | Web 版 |
|------|----------------|--------|
| 部署方式 | 独立 exe 文件 | Web 服务器 |
| 后端启动 | 自动启动/关闭 | 需要手动管理 |
| 数据库 | 本地文件 | 服务器文件 |
| 访问方式 | 双击运行 | 浏览器访问 |
| 更新方式 | 替换 exe | 重新部署 |
| 适用场景 | 单机使用 | 多人协作 |

## 常见问题

### 1. 前端无法连接后端

检查：
- 后端是否正常运行（访问 http://localhost:8000/docs）
- API 地址配置是否正确
- CORS 配置是否正确

### 2. 端口被占用

修改端口：
- 后端：修改 `backend/main.py` 中的 `port` 参数
- 前端：使用其他端口启动 HTTP 服务器

### 3. 数据库文件位置

- 开发环境：`backend/database.db`
- 生产环境：建议放在独立的数据目录

## 安全建议

1. 生产环境使用 HTTPS
2. 限制 CORS 允许的来源
3. 配置防火墙规则
4. 定期备份数据库
5. 使用环境变量管理敏感配置
6. 配置访问日志和错误日志

## 性能优化

1. 使用 Nginx 缓存静态资源
2. 启用 Gzip 压缩
3. 配置 CDN（可选）
4. 数据库定期优化
5. 监控服务器资源使用
