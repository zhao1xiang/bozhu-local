# 眼科注射预约系统 - 后端服务部署说明

## 📋 环境信息
- **域名**: local-show.microcall.com.cn
- **后端端口**: 8031
- **前端路径**: /work/local-show/frontend/
- **后端路径**: /work/local-show/backend/
- **Python版本**: Python 3.6+

## 🚀 快速部署步骤

### 步骤1：上传文件到服务器
```bash
# 上传整个 web-dist 目录到服务器
scp -r web-dist root@your_server_ip:/tmp/
```

### 步骤2：SSH登录服务器并部署
```bash
ssh root@your_server_ip

# 创建部署目录
mkdir -p /work/local-show

# 复制文件
cp -r /tmp/web-dist/* /work/local-show/

# 进入后端目录
cd /work/local-show/backend
```

### 步骤3：启动后端服务

#### 方法1：使用完整管理脚本（推荐）
```bash
# 给脚本执行权限
chmod +x ../start-backend-service.sh

# 安装依赖并启动
sudo bash ../start-backend-service.sh start
```

#### 方法2：使用快速启动脚本
```bash
# 给脚本执行权限
chmod +x ../start-backend.sh

# 启动服务
sudo bash ../start-backend.sh
```

#### 方法3：手动启动
```bash
# 安装依赖
pip3 install -r requirements.txt

# 创建日志目录
mkdir -p logs

# 启动服务（端口8031）
nohup python3 run_server.py --port 8031 > logs/backend.log 2>&1 &
```

## 🔧 服务管理命令

### 使用完整管理脚本
```bash
# 启动服务
sudo bash /work/local-show/start-backend-service.sh start

# 停止服务
sudo bash /work/local-show/start-backend-service.sh stop

# 重启服务
sudo bash /work/local-show/start-backend-service.sh restart

# 查看状态
sudo bash /work/local-show/start-backend-service.sh status

# 查看日志
sudo bash /work/local-show/start-backend-service.sh logs

# 安装为 systemd 服务（开机自启）
sudo bash /work/local-show/start-backend-service.sh install
```

### 手动管理
```bash
# 查看进程
ps aux | grep run_server.py

# 查看端口占用
netstat -tuln | grep 8031

# 停止服务
pkill -f "run_server.py"

# 查看日志
tail -f /work/local-show/backend/logs/backend.log
```

## 🔍 故障排查

### 问题1：端口被占用
```bash
# 查看占用进程
lsof -i :8031

# 或
netstat -tuln | grep 8031

# 停止占用进程
kill <PID>
```

### 问题2：Python依赖错误
```bash
# 重新安装依赖
cd /work/local-show/backend
pip3 install --upgrade pip
pip3 install -r requirements.txt
```

### 问题3：服务无法启动
```bash
# 查看详细日志
tail -50 /work/local-show/backend/logs/backend.log

# 检查 Python 版本
python3 --version

# 检查文件权限
ls -la /work/local-show/backend/
```

### 问题4：Nginx代理失败
```bash
# 检查 Nginx 配置
nginx -t

# 查看 Nginx 错误日志
tail -f /var/log/nginx/error.log

# 重启 Nginx
systemctl restart nginx
```

## ⚙️ Nginx 配置要点
确保你的 Nginx 配置包含以下内容：
```nginx
location @api_proxy {
    proxy_pass http://127.0.0.1:8031;
    proxy_set_header Host $host;
    proxy_set_header X-Real-Ip $remote_addr;
    proxy_set_header X-Forwarded-Proto https;
    proxy_set_header X-Forwarded-For $remote_addr;
    proxy_set_header Connection "keep-alive";
    proxy_http_version 1.1;
}
```
