# 数据库自动迁移说明

## 概述
眼科注射预约系统 v2.2.3 引入了自动数据库迁移功能，解决版本升级时的数据库兼容性问题。

## 自动迁移功能

### 启动时自动检测
- 系统启动时会自动检测数据库结构
- 如发现字段类型不匹配，会自动进行迁移
- 特别针对视力字段的数据类型优化

### 支持的迁移类型
1. **视力字段类型迁移**
   - 从 REAL 类型迁移到 TEXT 类型
   - 保持数据完整性，支持更灵活的视力记录

2. **新字段添加**
   - 自动添加新版本引入的字段
   - 设置合理的默认值

3. **数据格式转换**
   - 自动处理数据格式变更
   - 确保历史数据兼容性

## 迁移过程

### 自动迁移流程
1. 系统启动时检测数据库结构
2. 对比当前版本要求的结构
3. 自动执行必要的迁移操作
4. 记录迁移日志

### 迁移日志
迁移过程会在控制台和日志文件中记录：
```
[INFO] 检查数据库结构...
[INFO] 发现视力字段需要迁移...
[INFO] 正在迁移视力字段类型...
[INFO] 数据库迁移完成
```

## 手动修复脚本

如果自动迁移失败，可以使用手动修复脚本：

### 使用方法
```bash
cd backend
python3 fix_database_schema.py
```

### 脚本功能
- 检测并修复数据库结构问题
- 处理视力字段类型转换
- 验证数据完整性
- 生成详细的修复报告

## 版本兼容性

### 支持的版本升级路径
- v2.1.x → v2.2.3 ✓
- v2.2.0 → v2.2.3 ✓
- v2.2.1 → v2.2.3 ✓
- v2.2.2 → v2.2.3 ✓

### 数据保护
- 迁移前自动备份关键数据
- 支持回滚到迁移前状态
- 保持数据完整性和一致性

## 故障排除

### 常见问题

#### 1. 迁移失败
**症状**：启动时报告迁移失败
**解决**：
```bash
# 运行手动修复脚本
python3 fix_database_schema.py

# 如果仍有问题，检查数据库文件权限
chmod 644 database.db
```

#### 2. 字段类型错误
**症状**：查询时出现类型转换错误
**解决**：
```bash
# 强制重新迁移视力字段
python3 -c "
from auto_migrate import migrate_vision_fields_to_text
migrate_vision_fields_to_text('database.db')
"
```

#### 3. 数据丢失担心
**解决**：
```bash
# 手动备份数据库
cp database.db database.db.backup_$(date +%Y%m%d_%H%M%S)

# 验证数据完整性
python3 -c "
import sqlite3
conn = sqlite3.connect('database.db')
cursor = conn.cursor()
cursor.execute('SELECT COUNT(*) FROM patient')
print(f'患者记录数: {cursor.fetchone()[0]}')
conn.close()
"
```
