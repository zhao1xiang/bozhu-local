# 数据库独立性修复说明

## 问题描述

之前替换 `dist-package\backend\database.db` 文件后不生效，新增的账号无法登录，清空的数据仍然存在。

## 根本原因

Tauri 配置中的资源打包设置：
```json
"resources": {
  "../../backend/dist/*": "./"
}
```

这个配置会将 `backend/dist/` 下的**所有文件**（包括 `database.db`）打包到 EXE 中。每次启动应用时，使用的是打包时的数据库快照，而不是磁盘上的文件。

## 解决方案

### 1. 修改 Tauri 配置

修改 `frontend/src-tauri/tauri.conf.json`：
```json
"resources": {
  "../../backend/dist/backend_server.exe": "./"
}
```

**只打包 backend_server.exe**，不打包数据库文件。

### 2. 重新打包

```bash
# 增加 Node.js 内存限制
$env:NODE_OPTIONS="--max-old-space-size=4096"

# 构建前端
cd frontend
npm run build

# 打包 Tauri
npm run tauri build

# 复制到分发目录
Copy-Item "src-tauri\target\release\app.exe" "..\dist-package\眼科注射预约系统.exe" -Force
```

## 重要说明

✅ **现在可以做的：**
- 随时替换 `dist-package\backend\database.db`
- 重启应用后立即生效
- 不需要重新打包整个应用

❌ **之前不能做的：**
- 替换数据库文件不生效
- 数据被"锁定"在 EXE 中
- 必须重新打包才能更新数据

- app.exe：11.2 MB（前端 + Tauri 运行时）

### 内存优化
由于前端资源较大，需要增加 Node.js 内存限制：
```powershell
$env:NODE_OPTIONS="--max-old-space-size=4096"
```
