# 使用阿里云云效自动打包Mac版本

## 什么是阿里云云效？

阿里云云效是阿里云提供的DevOps平台，类似GitHub Actions，可以在云端自动打包应用。

## 优势

✅ **与CodeUp集成**（你的代码已经在CodeUp上）
✅ **支持Mac构建环境**
✅ **国内访问速度快**
✅ **有免费额度**

## 使用步骤

### 1. 开通云效服务

1. 访问：https://devops.aliyun.com/
2. 使用阿里云账号登录
3. 创建企业（如果还没有）

### 2. 创建流水线

1. 进入"流水线" -> "新建流水线"
2. 选择"自定义流水线"
3. 配置流水线：

#### 代码源配置
- 代码源：选择你的CodeUp仓库
- 分支：master
- 触发方式：手动触发 或 代码提交时触发

#### 构建环境
- 构建环境：macOS（选择Mac构建机）
- Python版本：3.9
- Node.js版本：18

#### 构建脚本
```bash
#!/bin/bash
set -e

echo "=========================================="
echo "开始构建Mac版本"
echo "=========================================="

# 1. 构建前端
echo "1. 构建前端..."
cd frontend
npm install
npm run build
cd ..

# 2. 安装后端依赖
echo "2. 安装后端依赖..."
cd backend
pip3 install -r requirements.txt
pip3 install pyinstaller

# 3. 打包后端
echo "3. 打包后端..."
pyinstaller build_mac.spec --clean --noconfirm

# 4. 创建发布包
echo "4. 创建发布包..."
cd ..
mkdir -p simple-web-package-mac-v2.1.8
cp backend/dist/backend_server simple-web-package-mac-v2.1.8/
chmod +x simple-web-package-mac-v2.1.8/backend_server
cp -r frontend/dist simple-web-package-mac-v2.1.8/frontend

# 创建启动脚本
cat > simple-web-package-mac-v2.1.8/启动服务.sh << 'EOF'
#!/bin/bash
SCRIPT_DIR="$( cd "$( dirname "${BASH_SOURCE[0]}" )" && pwd )"
cd "$SCRIPT_DIR"
./backend_server
EOF
chmod +x simple-web-package-mac-v2.1.8/启动服务.sh

# 创建说明文件
cat > simple-web-package-mac-v2.1.8/使用说明.txt << 'EOF'
玻注预约系统 - Mac版 v2.1.8

使用方法：
1. 双击运行 "启动服务.sh"
2. 浏览器会自动打开系统页面
3. 使用 admin/admin123 登录

如有问题，请查看项目文档。
EOF

# 5. 创建ZIP
echo "5. 创建ZIP..."
zip -r simple-web-package-mac-v2.1.8.zip simple-web-package-mac-v2.1.8/

echo "=========================================="
echo "构建完成！"
echo "=========================================="
ls -lh simple-web-package-mac-v2.1.8.zip
```

#### 产物配置
- 产物路径：`simple-web-package-mac-v2.1.8.zip`
- 产物名称：`mac-package-v2.1.8`

### 3. 运行流水线

1. 保存流水线配置
2. 点击"运行"按钮
3. 等待构建完成（约10-15分钟）
4. 在"产物"中下载打包好的ZIP文件

## 费用说明

阿里云云效的收费：
- **免费版**：每月有一定的构建时长
- **企业版**：按需付费

Mac构建机可能需要付费，具体价格请查看云效官网。

## 替代方案：使用Linux构建机 + 源码包

如果Mac构建机太贵，可以：

1. 使用Linux构建机（更便宜）
2. 只构建前端
3. 提供源码运行包给Mac用户

构建脚本：
```bash
#!/bin/bash
set -e

# 只构建前端
cd frontend
npm install
npm run build
cd ..

# 打包源码
mkdir -p mac-source-package
cp -r backend mac-source-package/
cp -r frontend/dist mac-source-package/backend/frontend
cp Mac源码运行指南.md mac-source-package/README.md

# 创建启动脚本
cat > mac-source-package/start.sh << 'EOF'
#!/bin/bash
cd backend
pip3 install -r requirements.txt
python3 simple_web_server.py
EOF
chmod +x mac-source-package/start.sh

# 打包
tar -czf mac-source-package.tar.gz mac-source-package/
```

## 常见问题

### Q: 云效支持Mac构建吗？
A: 支持，但可能需要付费。建议先咨询阿里云客服。

### Q: 构建失败怎么办？
A: 查看构建日志，找到错误信息。常见问题：
   - 依赖安装失败
   - 权限问题
   - 环境配置问题

### Q: 能自动发布吗？
A: 可以，云效支持：
   - 自动上传到OSS
   - 自动部署到服务器
   - 发送钉钉通知

## 总结

使用阿里云云效的优势：
- ✅ 与CodeUp无缝集成
- ✅ 国内访问速度快
- ✅ 支持Mac构建

但需要注意：
- ❌ Mac构建机可能需要付费
- ❌ 配置相对复杂

如果预算有限，建议使用"源码运行包"方案。
